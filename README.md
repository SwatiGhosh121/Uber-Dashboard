Uber Trips Data Analysis Dashboard

Project Overview
This project analyzes Uber trip data (50,000 records) to understand travel patterns, operational efficiency, pricing behavior, and customer usage trends.
The analysis is visualized through interactive dashboards built using Tableau.

Objective
The goal of this project is to:
Identify peak demand hours
Analyze trip completion and cancellation patterns
Understand fare and distance trends
Improve operational decision-making using data insights

Dataset Description
The dataset contains:
Trip ID
Driver ID & Rider ID
City
Pickup & Drop coordinates
Pickup & Drop timestamps
Distance covered
Fare amount
Trip status (Completed / Cancelled / No-show)
Payment method
Dashboard Structure
Page 1: Business Performance Overview
Provides a high-level summary of Uber operations.
Key Visuals:
Total Trips
Total Revenue
Average Fare
Average Distance per Trip
Trip Status Distribution
Trips by Time (Hourly Trend)
Trips by City
Insights:
Identifies peak demand hours
Highlights top-performing cities
Shows overall service efficiency

Page 2: Deep Dive Analysis
Focuses on detailed operational and behavioral insights.
Key Visuals:
Cancelled Trips by Hour
Average Trip Duration by Hour
Average Fare by Hour
Distance Distribution
Insights:
Detects peak cancellation periods
Reveals traffic patterns through trip duration
Shows surge pricing behavior
Highlights short vs long trip trends
Key Insights
Peak demand occurs during morning and evening hours
Higher cancellations during peak hours indicate supply-demand imbalance
Trip duration increases during busy periods due to traffic
Fare increases during high demand, reflecting dynamic pricing
Majority of trips are short-distance, indicating intra-city travel
